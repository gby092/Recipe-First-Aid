# Recipe First Aid · 菜谱急救箱

> 一个 AI 辅助开发的跨平台移动应用 — 输入你冰箱里剩下的食材，立刻告诉你今晚能做什么菜。

## 项目解决的核心痛点

许多独居用户打开冰箱后面对零散的食材（比如一个鸡蛋、半根黄瓜、剩饭）不知道做什么，最后点外卖。现有菜谱 App 都是“按菜名搜食材”，而不是“按食材找菜谱”。

## 核心逻辑流（AI Agent 设计）

本项目基于 **Cursor + Claude 3.5 Sonnet** 进行开发，包含以下 AI 驱动的推理流程：

1. **长链推理**  
   - 用户输入 2~3 种食材（如“鸡蛋、剩饭、青椒”）  
   - Agent 将名称归一化（“青椒” = “辣椒”）  
   - 检索本地菜谱库（20个内置菜谱），计算匹配度：完全匹配 > 缺1种辅料 > 缺2种辅料  
   - 若完全不匹配，推理出可选的替代食材（例如“没有青椒，可以用黄瓜做蛋炒饭”）

2. **多 Agent 协作**  
   - Agent A：食材分析与推荐排序  
   - Agent B：步骤生成与简化（将原菜谱压缩为 3 步以内的新手友好版本）  
   - 两者通过共享 JSON 状态交换中间结果，最终输出菜谱卡片

## 技术栈

- **前端框架**：Flutter（一套代码运行于 iOS / Android）  
- **本地存储**：shared_preferences  
- **AI 辅助开发**：Cursor + Claude 3.5 Sonnet  
- **鸿蒙版本**：计划用 DevEco Studio 编译为鸿蒙应用
